Upload Workflow

When to export: upload one Sentral absence export after Period 2 and before second break, then upload the full-day export again after Period 5.

Sentral path: Sentral -> School Attendance -> Reports -> Absence Lists -> Export

File type: upload the Sentral .xls absence export.

How the app now decides the upload type: the backend infers whether the file is the earlier upload or the end-of-day upload by checking the report date, any earlier upload already stored for that date, the current Sydney time, and the number of data rows compared with the earlier file.

How lateness is identified: the parser only treats missed roll-call rows as late-to-school cases. It looks for Shorthand / Description combinations of U + Unjustified or ? + Absent, and requires the left side of the Time range to start at 8:00AM or 8:25AM.

Arrival time and lateness: the code splits the Time value on -, uses the right-hand side as the arrival time, and measures lateness against 8:35AM.

Detention flow: students found in the earlier upload can be scheduled for same-day second break, while students only found in the end-of-day upload are carried to the next school day.
