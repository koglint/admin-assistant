Upload Workflow

Use this page whenever you want to refresh the system with a new Sentral export. You can upload multiple reports for the same day as more information becomes available.

Upload File

When to export: Export the Sentral Report AFTER Period 2 and BEFORE Lunch/ second break. THis must be done every day.

Sentral path: Sentral -> School Attendance -> Reports -> Absence Lists -> Export

File type: upload the Sentral .xls absence export.

No upload mode selection is needed: the backend reads the attendance rows inside the spreadsheet itself and merges any new information into the existing student records for that report date.

How lateness is identified: the parser only treats missed roll-call rows as late-to-school cases. It looks for Shorthand / Description combinations of U + Unjustified or ? + Absent, and requires the left side of the Time range to start at 8:00AM or 8:25AM.

Arrival time and lateness: the code splits the Time value on -, uses the right-hand side as the arrival time, and measures lateness against 8:35AM.

Detention flow: same-day or next-day detention is decided from the student's actual arrival time in the file. If they arrived before first break, they can be scheduled for same-day second break. If they arrived after first break, they are carried to the next school day.

Detention absence checks: if a student was marked absent from detention, later uploads for that same report date are used to work out whether they were actually at school or absent for the day.
